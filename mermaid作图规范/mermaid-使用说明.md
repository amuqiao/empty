# Mermaid 作图规范 · 使用说明

共 4 份文档，按分析场景选用，每次只挂载一份。

---

## 选哪份

```
想理解系统结构和业务流程      → 挂载  mermaid-A-系统认知层.md
想深入模块的数据和代码设计    → 挂载  mermaid-B-代码深潜层.md
想分析异步链路或实体状态      → 挂载  mermaid-C-运行时行为层.md
想规划项目、梳理演进或总览依赖 → 挂载  mermaid-D-规划可视化层.md
```

| 文件 | 包含图表 | 典型触发词 |
|------|---------|----------|
| A 系统认知层 | 系统架构图、端到端流程图 | "画架构图""梳理业务流程""分析这个项目" |
| B 代码深潜层 | 数据模型关系图、类层级关系图 | "分析表结构""梳理类的继承关系""理解这个模块" |
| C 运行时行为层 | 状态机图、时序图 | "分析 Celery 链路""画状态机""SSE 流程" |
| D 规划可视化层 | 甘特图、时间线图、思维导图 | "画项目路线图""技术演进史""总览技术栈" |

---

## 怎么用

在提示词中加一句：

> Mermaid 作图风格参考 `@mermaid-A-系统认知层.md`，严格遵循其中的配色、结构和规范。

需要多种图时，同时挂载多份文档即可，文件之间无冲突。

---

## 注意

文件 C 的两种图（`stateDiagram-v2` 和 `sequenceDiagram`）使用与 A/B 完全不同的 Mermaid 语法，不支持 `classDef`、`linkStyle` 等 flowchart 专属语法，挂载时不要与 A/B 混用规则。

文件 D 的三种图（`gantt`、`timeline`、`mindmap`）同样不支持 flowchart 专属语法；其中 mindmap 节点换行用 `<br/>`（含闭合斜杠），与 A/B 的 `<br>` 写法不同。
